***Forkify Functional Testing (Manual QA Project)***

This project demonstrates **manual functional testing** of the Forkify recipe web application. The application allows users to search, view, bookmark, and upload recipes using an external API.

The primary goal of this project is to validate that all core features and user workflows of the application function correctly according to the requirements. Functional testing was conducted from an end-user perspective, focusing on real-world scenarios such as searching recipes, viewing details, adjusting servings, bookmarking, and uploading new recipes.

### 🔍 Testing Scope

* Search functionality and result validation
* Recipe details display (ingredients, cooking time, servings)
* Increase/decrease servings and dynamic updates
* Bookmark feature and local storage behavior
* Add/upload recipe functionality
* Input validation and error handling
* End-to-end user workflows


### 🧪 Testing Approach

* Designed and executed **manual test cases and test scenarios**
* Verified **input/output behavior and system responses**
* Covered **positive, negative, and edge test cases**
* Ensured smooth **user interaction and workflow validation**

Functional testing ensures that each feature behaves as expected and meets business requirements by validating inputs, outputs, and user interactions. ([BrowserStack][2])

This project helped identify functional issues, validate application behavior, and ensure a smooth user experience across key features.
